# 技术实现改进说明

## 已完成内容

本轮实现了三类技术改进：Beta 记忆机制、规则冲突检测与解决、规则一致性验证。

## Beta 记忆机制

系统原先主要使用 Alpha 索引，即通过输入事实找到候选规则。现在补充了 Beta 记忆，用于记录每条规则已经满足和未满足的条件，避免在多轮推理中重复检查没有变化的条件。

实验输出：

- `results/beta_ablation_summary.csv`

当前结果显示：

- Alpha-only 平均条件检查次数：49.7；
- Alpha+Beta 平均条件检查次数：17.69；
- 条件检查减少比例：约 64.41%；
- Alpha+Beta 平均耗时高于 Alpha-only，说明 Python 层维护 Beta 缓存存在额外开销。

论文建议表述：

> Beta 记忆机制显著减少重复条件检查，但在当前 Python 实现中，缓存维护开销抵消了部分速度收益。因此本文将其作为完整 Rete 思想的补充验证，并明确说明存在内存与速度权衡。

## 规则冲突检测与解决

新增运行时冲突检测：当一次推理同时推出互斥大类时，系统会记录冲突。

互斥大类包括：

- 哺乳动物
- 鸟类
- 鱼类
- 爬行动物
- 昆虫
- 两栖动物

默认解决策略：

- 若冲突结论优先级不同，采用优先级最高者；
- 若优先级相同，保留多个结论并标记为人工审查。

实验输出：

- `results/runtime_conflict_log.csv`

## 规则一致性验证

新增离线规则验证，主要检查：

- 相同条件集对应不同结论的离线冲突；
- 完全重复规则；
- 弱冗余规则；
- 规则依赖图中的循环依赖；
- 规则库健康度。

实验输出：

- `results/conflict_report.csv`
- `results/rule_consistency_report.json`

当前规则库健康度：

- 规则总数：5121；
- 离线冲突数：45；
- 完全重复规则数：0；
- 弱冗余规则数：0；
- 循环依赖数：0；
- 健康度分数：99.12。

论文建议表述：

> 一致性验证发现当前规则库不存在循环依赖和重复规则，但生成式扩展规则中存在少量同条件集对应不同结论的冲突规则对。该结果为后续规则库人工审查提供了依据。
